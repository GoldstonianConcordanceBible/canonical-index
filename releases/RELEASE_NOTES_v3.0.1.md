# v3.0.1 — Canonical Integrity Patch

This patch release aligns repository structure, documentation, and machine-readable expectations after the v3.0.0 Canonical Intelligence Layer release.

## What this patch does

- adds explicit governance and version metadata
- clarifies the purpose of similarly named directories
- reduces trust ambiguity for agents, contributors, and researchers
- tightens the mapping between release claims and repository structure

## Key outcomes

- clearer separation between canonical data and legacy/supporting materials
- clearer separation between machine-readable evals and evaluation documentation
- improved release hygiene and discoverability
- improved contributor guidance and authority order
