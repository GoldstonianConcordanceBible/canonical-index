# GCB Ecosystem Architecture

The **Goldstonian Concordance Bible (GCB)** ecosystem is organized as a layered scripture intelligence architecture built on the **81-Book Ethiopian Canon** and interpreted through the **Mirror → Water → Fire Doctrine**.

This document explains how the major layers of the ecosystem interact.

---

# Layer 1 — Canonical Registry

Location:

canon/books/index.json

Role:

Identity authority for biblical books.

Responsibilities:

- define canonical order
- preserve Ethiopian canon distinctions
- provide stable book identifiers

---

# Layer 2 — Knowledge Graph

Location:

kg/universal_graph.json

Role:

Graph structure linking:

- books
- characters
- events
- themes
- doctrine nodes

This layer enables scripture connections across the canon.

---

# Layer 3 — Ontology

Role:

Defines semantic classes and relationships used in the graph.

Examples:

- book
- character
- event
- doctrine
- theme

---

# Layer 4 — Agent Layer

Agents interpret the canonical registry and graph using structured rules.

Agent roles include:

- canon resolution
- scripture retrieval
- doctrine interpretation
- skeptic engagement

---

# Layer 5 — Discovery Layer

Discovery components expose the ecosystem to:

- search engines
- AI crawlers
- knowledge graph systems
- research indexers

---

# Architecture Goal

The goal of this layered structure is to treat the **Bible not only as text but as a structured knowledge system** rooted in the Ethiopian canon.
